解密@SpringBootApplication启动原理

一、本课程目标

学习spring boot的核心注解@SpringBootApplication，掌握@SpringBootApplication的原理。

二、剖析@SpringBootApplication的源码

首先我们分析springboot的启动注解@SpringBootApplication

  1. @Target(ElementType.TYPE) 注解目标位置：接口、类、枚举
  2. @Retention(RetentionPolicy.RUNTIME) 注解会在class字节码文件中存在，在运行时可以通过反射获取到
  3. @Documented 用于生成javadoc,默认情况下，javadoc是不包括注解的，但是如果声明注解时指定了@Documented，则他会被javadoc之类的工具类处理，所以直接类型信息也会被包括在生成的文档中
    @Inherited，在继承关系中，如果子类继承父类的注解，那么要该注解必须被@Inherited修饰的注解

除了以上常规的注解，剩下几个就是springboot的核心注解了.@SpringBootApplication就是一个复合注解包含了SpringBootConfiguration、EnableAutoConfiguration和ComponentScan

三、@SpringBootApplication只干了一件事

以上所有注解就只干一件事：把bean注册到spring ioc容器中。
通过3种方式来实现：

  1. @SpringBootConfiguration 通过@configuration与@Bean结合，注册到Springioc容器。
  2. @ComponentScan 通过范围扫描的方式，扫描待定注解类，将其注册到springioc容器
  3. @EnableAutoConfiguration通过spring.factories的配置，来实现bean的注册。

四、课后练习题

这节课，通过解读@SpringBootApplication ，知道他是一个复合注解。
新建一个springboot项目，然后清除@SpringBootApplication ，用@SpringBootConfiguration、@ComponentScan和@EnableAutoConfiguration代替，然后启动springboot,看能不能成功
